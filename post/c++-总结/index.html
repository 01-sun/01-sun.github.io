抽象类和接口

抽象类:含纯虚函数

接口: 都是纯虚函数

纯虚函数: 父类定义，由子类实现

单例模式

Why:避免频繁使用的类被频繁创建释放

What:就是自行创建唯一实例，并向整个系统提供这个实例

写法:


class A{
    public:
    static A *instance;
    static A* getInstance(){
        if(instance==nullptr)  instance = new A();
        returhn instance;
    }
    private: 
        A();
        A(const A&);
        A& oprator=(const A&);

}

注意：单例应该只用来保存全局的状态，并且不能和任何作用域绑定。如果这些状态的作用域比一个完整的应用程序的生命周期要短，那么这个状态就不应该使用单例来管理。

大小端

小端:低地址低字节，使用内存运算

大端:低地址高字节，适用于数据运算，快速判断数据正负大小

超前引用

What:类A和类B需要彼此互相引用，这样必然有一个类A会先被定义，而另外一个类B后被定义，这样在先被定义的类引用后被定义的类的时候，就导致了所谓的超前引用。

How:

  • 用之前先声明class B

  • 使用全局变量 extern

const用法

目的:常量不让改变

const char str[] = “1234”; //const修饰str的值，不允许值被修改

const char* str == “1234”: //同上

char* const str= “1234”: //const修饰指针,指针不能指向其他

const存储位置：

  • 修饰局部变量，存储在栈。

  • 修饰字面值或全局变量，存储在全局静态存储区。

注意

const修饰成员函数，实际上修饰this指针，表明在该成员函数中不能对类的任何成员进行修改。

并且不能与static同时使用，静态方法不依赖类的实例

lambda

目的：匿名函数只要在调用时才创建空间,调用结束立刻释放,节省空间。而普通函数定义时占内存

What: 没有名字的函数 type{}格式，type:=值类型 &引用类型
